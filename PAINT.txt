塗りつぶし

== 書式 ==
<!--

  ・[A]    Aを省略可
  ・<A|B>  AまたはBを指定

-->

★ 書式に省略可能な繰り返し [,引数...] が含まれている場合は、{,引数} に変更してください。
; 書式1
: PAINT ( X , Y ) [ , 描画色 ] [ , 境界色 ]
; 書式2
: PAINT ( X , Y ) [ , 描画パターン ] [ , 境界色 ] [ , 背景パターン ]

== 解説 ==

★なるべく、ですます調に変更してください。分量が多い場合はそのままで構いません。


座標(X,Y)を含み、境界色(パレット番号)で囲まれた領域を塗りつぶす。 書式1は単色で、書式2は模様(パターン)で塗りつぶす。

STEP(X,Y) の形式(相対座標)で指定してもよい(POINT(最終描画位置設定)参照)。 座標はワールド座標系を使用する。

境界色省略時は、領域色と同じと見なす。

描画色(パレット番号)は、塗りつぶす色を指定。省略時はCOLORで指定した描画色。

描画パターンは塗つぶす模様を指定する。模様は、幅8ドット×高さnドット(nは1以上で任意)の ビットマップ。これを、次の方法で最終的に文字列とし、描画パターンに指定する。
; 白黒モードの場合
: 1ドットを1 bitで表現。「白」は1、「黒」は0。各ライン(8ドット)は8 bitで表される。 この8 bitを文字コードとする文字1文字を1ライン分の文字とし、 nライン分結合したものを描画パターンとして指定する。
<pre>
1ライン目 ■□□□■□■□ ⇒ 2進数で10001010、16進数で8A
2ライン目 □□□■□■□■ ⇒ 2進数で00010101、16進数で15
3ライン目 □□■□■□■□ ⇒ 2進数で00101010、16進数で2A
4ライン目 □■□■□■□□ ⇒ 2進数で01010100、16進数で54
   ↓
CHR$(&h8A) + CHR$(&h15) + CHR$(&h2A) + CHR$(&h54)
</pre>
; カラーモードの場合
: 1ドットごとにパレット番号(0～7)で色を指定できる。1ドットをパレット番号(0～7)の3 bitで表現することになり、 各ラインは、8ドット×3 bit=24 bit=3バイトで表されることになる。 1ドット分のパレット番号(3 bit)は、この3バイトの間で1bitずつに分解して持たせる。 すなわち、1バイト目はパレット番号のbit 0だけを持ち、2バイト目はbit 1、3バイト目はbit 2だけを持つ。 そして、各バイトの各々のbitは、模様の1ドットにそれぞれ対応し、8 bitで8ドット分を表す。 この8 bitを文字コードとする文字を3バイト分結合して3文字の文字列を得て、それを1ライン分の文字列とする。 nライン分必要な場合は、以上のように作成した3文字を必要なライン数分結合する。
<pre>
1ライン8ドットのパレット番号 [1][6][1][7][1][6][1][3]
        ↓
1バイト目:パレット番号のbit0  1  0  1  1  1  0  1  1  ⇒ 16進数で&hBB
2バイト目:パレット番号のbit1  0  1  0  1  0  1  0  1  ⇒ 16進数で&h55
3バイト目:パレット番号のbit2  0  1  0  1  0  1  0  0  ⇒ 16進数で&h54
        ↓
CHR$(&hBB) + CHR$(&h55) + CHR$(&h54)
</pre>

背景パターンは、補助的な情報で、塗りつぶし対象の領域が既に模様で塗りつぶされている場合に、そのパターンを指定する。 このパターンは幅8ドット×高さ1ドット。省略した場合、塗りつぶしが中断する場合がある。 また、このパターンが描画パターンに3回以上連続して出現する場合はエラーとなる。 (★指定内容詳細不明につき情報求む)

塗りつぶし対象の領域が複雑な形状の場合はOut of memoryエラーとなる場合がある。
